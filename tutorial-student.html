Make Your Game Playable by Everyone

Learn how to build a car-dodging game in Python and make it accessible — so anyone can play it, no matter their abilities.

What Are We Building?

We're going to build a simple game where you drive a car and dodge enemies coming at you. Then we'll make it accessible — meaning people with different abilities (like low vision, color blindness, or hearing differences) can enjoy it too.

Why does accessibility matter?

About 1 in 4 adults in the US has some type of disability. If your game only works for people with perfect vision and fast reflexes, you're leaving out a LOT of players. Accessible design makes games better for everyone — not just disabled people. Ever wanted a "slow mode" to practice? That's an accessibility feature!

What You'll Need

  • A computer with Python installed
  • A text editor (VS Code, IDLE, or anything you like)
  • Pygame (we'll install it below)
  • About 2 hours of time

Quick Vocab

Pygame
A free Python tool for making games with graphics and sound
WCAG
Rules (from the W3C organization) about making digital stuff usable by everyone
Sprite
A game character or object (your car, the enemy car, etc.)
FPS
Frames per second — how many times the screen redraws. Higher = smoother and faster
Contrast
How different two colors look next to each other. High contrast = easy to see
1

Setting Up

First, let's install Pygame. Open your terminal (or command prompt on Windows) and type:

pip install pygame

Now create a new file called game.py and add these first lines:

# These lines load the tools we need
import pygame, sys
from pygame.locals import *
import random, time

# Start up Pygame
pygame.init()
What's happening here?

import pygame loads the game-making tools. pygame.init() turns them on. Think of it like plugging in a game console and pressing the power button — you need both steps.

2

Creating the Game Window

Next, let's set up our game window and some basic settings:

# Game speed settings
normal_mode = 60   # Normal speed: 60 frames per second
slow_mode = 30     # Slow speed: 30 frames per second
FPS = normal_mode
FramePerSec = pygame.time.Clock()

# Colors (Red, Green, Blue) - each value from 0 to 255
BLUE  = (0, 0, 255)
RED   = (255, 0, 0)
GREEN = (0, 255, 0)
BLACK = (0, 0, 0)
WHITE = (255, 255, 255)

# Game window size
SCREEN_WIDTH = 400
SCREEN_HEIGHT = 600
SPEED = 5
SCORE = 0

# Create the window
DISPLAYSURF = pygame.display.set_mode((SCREEN_WIDTH, SCREEN_HEIGHT))
DISPLAYSURF.fill(WHITE)
pygame.display.set_caption("My Game")
Colors are just 3 numbers

Every color on your screen is made from mixing Red, Green, and Blue light. Each goes from 0 (none) to 255 (maximum). So (255, 0, 0) is pure red, (0, 0, 0) is black (no light), and (255, 255, 255) is white (all the light). Try making your own colors!

3

Creating Your Characters

In games, we use "classes" to create characters. Think of a class as a cookie cutter — it's a template, and every cookie (game object) made from it has the same shape but can be decorated differently.

The Enemy (the car you dodge)

class Enemy(pygame.sprite.Sprite):
    def __init__(self):
        super().__init__()
        self.image = pygame.image.load("Enemy.png")
        self.rect = self.image.get_rect()
        # Start at a random spot at the top of the screen
        self.rect.center = (random.randint(40, SCREEN_WIDTH-40), 0)

    def move(self):
        global SCORE
        # Move down the screen
        self.rect.move_ip(0, SPEED)
        # If the enemy goes past the bottom, respawn at the top
        if self.rect.top > 600:
            SCORE += 1
            self.rect.top = 0
            self.rect.center = (random.randint(40, SCREEN_WIDTH - 40), 0)

The Player (you!)

class Player(pygame.sprite.Sprite):
    def __init__(self):
        super().__init__()
        self.image = pygame.image.load("Player.png")
        self.rect = self.image.get_rect()
        self.rect.center = (160, 520)    # Start near the bottom

    def move(self):
        pressed_keys = pygame.key.get_pressed()
        # Move left (but don't go off screen)
        if self.rect.left > 0:
            if pressed_keys[K_LEFT]:
                self.rect.move_ip(-5, 0)
        # Move right (but don't go off screen)
        if self.rect.right < SCREEN_WIDTH:
            if pressed_keys[K_RIGHT]:
                self.rect.move_ip(5, 0)
What's a "rect"?

A rect (short for rectangle) is an invisible box around your character. The game uses this box to figure out where your character is and whether it bumped into anything. Every sprite has one.

4

The Game Loop — Making It Run

The game loop is the heartbeat of your game. It runs over and over (60 times per second!) doing three things: check what happened, update everything, and draw it on screen.

# Create the characters
P1 = Player()
E1 = Enemy()

# Put them in groups (makes collision detection easier)
enemies = pygame.sprite.Group()
enemies.add(E1)
all_sprites = pygame.sprite.Group()
all_sprites.add(P1)
all_sprites.add(E1)

# Make the game get harder over time
INC_SPEED = pygame.USEREVENT + 1
pygame.time.set_timer(INC_SPEED, 1000)  # Every 1 second

# THE GAME LOOP
while True:

    # 1. CHECK WHAT HAPPENED
    for event in pygame.event.get():
        if event.type == INC_SPEED:
            SPEED += 0.5          # Game gets faster!
        if event.type == QUIT:
            pygame.quit()
            sys.exit()

    # 2. UPDATE EVERYTHING
    DISPLAYSURF.fill(WHITE)       # Clear the screen
    for entity in all_sprites:
        entity.move()
        DISPLAYSURF.blit(entity.image, entity.rect)

    # 3. CHECK FOR CRASHES
    if pygame.sprite.spritecollideany(P1, enemies):
        DISPLAYSURF.fill(RED)     # Flash red
        pygame.display.update()
        for entity in all_sprites:
            entity.kill()
        time.sleep(2)
        pygame.quit()
        sys.exit()

    # 4. DRAW AND WAIT
    pygame.display.update()
    FramePerSec.tick(FPS)
Try it!

Save and run your game with python game.py. You should see your car at the bottom and an enemy coming down from the top. Use the left and right arrow keys to dodge!

Note: You'll need image files called Player.png and Enemy.png in the same folder. Ask your teacher for these, or draw simple colored rectangles for now.

5

Wait... Not Everyone Can Play This

Your game works! But it has some serious problems that make it hard or impossible for some people to play:

Problem
Can you see this?

Blue text on grey — contrast ratio 1.07:1. Almost invisible for people with low vision.

Fixed
Easy to read!

Yellow on dark blue — contrast ratio 12.5:1. Clear for everyone.

Here's what's wrong and who it affects:

  • Low contrast colors: People with low vision or color blindness can't tell the player from the background
  • Tiny score text: Hard to read, especially for players with visual impairments
  • Only one speed: Too fast for players who need more reaction time (motor disabilities, cognitive disabilities, or just beginners!)
  • No volume control: Players with sensory sensitivities can't adjust the crash sound
  • Hard-to-read font: Some fonts look great but are awful for people with dyslexia or low vision
The rules we'll follow: WCAG AA

WCAG (Web Content Accessibility Guidelines) are rules created by accessibility experts for making digital stuff usable by everyone. The key rules for us:

  • Text must have a contrast ratio of at least 4.5:1
  • Game objects must have a contrast ratio of at least 3:1
  • Users should be able to control timing (speed)
  • Users should be able to control audio

You can check your contrast at webaim.org/resources/contrastchecker

Fix #1: Make Colors Pop

The score is almost invisible on the original game. Let's make it big, bold, and backed by a bright yellow box:

# After drawing the background, add this:
DISPLAYSURF.blit(background, (0, 0))

# Draw a bright yellow box behind the score
scores = font.render(str(SCORE), True, BLACK)
pygame.draw.rect(DISPLAYSURF, (255, 200, 0), (0, 0, 60, 65))
DISPLAYSURF.blit(scores, (15, 15))

We also need darker backgrounds and higher-contrast car sprites. If you're making your own sprites, use colors that look very different from each other — don't put a light blue car on a light grey road.

Check your contrast

Go to WebAIM's Contrast Checker. Type in your background color and your object color. Does it pass the 3:1 minimum for objects? Try different combos until you find ones that work!

7

Fix #2: Add a Slow Mode

Not everyone has the same reaction speed. Let's add a toggle so players can press Spacebar to switch between normal and slow mode:

# Add this variable before the game loop
mode_message_time = 0

# Inside the event loop, add this:
if event.type == KEYDOWN:
    if event.key == K_SPACE:
        if FPS == normal_mode:
            FPS = slow_mode         # Switch to slow
        else:
            FPS = normal_mode       # Switch back to normal
        mode_message_time = time.time()

# After drawing the score, show which mode we're in:
if time.time() - mode_message_time < 2:
    mode_text = "SLOW MODE" if FPS == slow_mode else "NORMAL MODE"
    mode_msg = font_medium.render(mode_text, True, BLACK)
    rect = mode_msg.get_rect(topleft=(110, 20))
    pygame.draw.rect(DISPLAYSURF, (255, 200, 0), rect)
    DISPLAYSURF.blit(mode_msg, rect)
How does this work?

Remember FPS (frames per second)? At 60 FPS, the game redraws 60 times per second. At 30 FPS, it redraws 30 times — half the speed. The game literally slows down because everything moves less often. The time.time() trick shows a "SLOW MODE" or "NORMAL MODE" message for 2 seconds after you press spacebar so the player knows what happened.

Fix #3: Use a Readable Font

The Atkinson Hyperlegible font was created by the Braille Institute specifically for people with low vision. It makes characters like I, l, and 1 look clearly different from each other.

# Download "Atkinson-Hyperlegible-Regular-102.ttf" and put it in your game folder
# Then set up your fonts like this:
special = "Atkinson-Hyperlegible-Regular-102.ttf"

font = pygame.font.SysFont(special, 60)          # Big - for "Game Over"
font_medium = pygame.font.SysFont(special, 40)    # Medium - for mode display
font_small = pygame.font.SysFont(special, 20)     # Small - for labels
game_over = font.render("Game Over", True, BLACK)

You can download this font for free at brailleinstitute.org/freefont.

Bad fonts vs. good fonts

Avoid: Pixelated fonts, cursive/script fonts, fonts with thin strokes, fonts where letters look the same (like I and l).

Good choices: Atkinson Hyperlegible, Verdana, Arial. These have clear, distinct characters and good spacing.

9

Fix #4: Let Players Control Volume

Some people are very sensitive to loud sounds. Others might not hear well. Let's give the player control using the up/down arrow keys:

# Add this variable at the top with your other variables
volume_level = 0.5  # Start at 50% volume

# In the Player's move() function, add:
def move(self):
    global volume_level
    pressed_keys = pygame.key.get_pressed()

    # Volume up (up arrow)
    if pressed_keys[K_UP] and volume_level < 1.0:
        volume_level = volume_level + 0.1
        volume_level = min(volume_level, 1.0)   # Don't go over 100%
    # Volume down (down arrow)
    if pressed_keys[K_DOWN] and volume_level > 0.0:
        volume_level = volume_level - 0.1
        volume_level = max(volume_level, 0.0)   # Don't go below 0%

    # ... rest of the move code (left/right) ...

# When the player crashes, play the sound at their chosen volume:
if pygame.sprite.spritecollideany(P1, enemies):
    crash = pygame.mixer.Sound('crash.wav')
    crash.set_volume(volume_level)
    crash.play()
    time.sleep(1)
    # ... game over code ...
What does min() and max() do?

min(volume_level, 1.0) picks whichever number is smaller. So if volume_level accidentally goes to 1.1, it gets capped back to 1.0. max() does the opposite — it prevents the volume from going below 0. This keeps the volume in a safe range.

!

You Did It!

Your game now has four accessibility features that the original didn't:

Your Accessibility Checklist
  • High-contrast colors — everyone can see the score and game objects clearly
  • Slow mode — players who need more time can press Spacebar to slow down
  • Readable font — Atkinson Hyperlegible is clear for people with low vision and dyslexia
  • Volume control — players can adjust sound to their comfort level

Want to do more?

  • Add color-blind mode: Use shapes (circles vs. squares) in addition to colors to tell objects apart
  • Add screen reader support: Look into the pyttsx3 library to announce score changes out loud
  • Let players choose their keys: Not everyone can use arrow keys easily
  • Add more enemies: Since we used classes, just do E2 = Enemy() and add it to the groups!
The big takeaway

Accessibility isn't about making a separate "special" version of your game. It's about making one game that works for as many people as possible. Every feature we added — slow mode, volume control, readable fonts — makes the game better for ALL players, not just disabled ones. That's what good design looks like.
